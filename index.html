📖 Tutorial & Dokumentasi API

🚀 Setup dalam 5 Menit

SeniorShield terdiri dari dua bagian: Frontend (dashboard ini) dan Backend API (server.js). Keduanya harus berjalan agar deteksi otomatis bekerja.

Alur Kerja Sistem: 📱 Android / SMS App │ POST /api/sms/incoming ▼ 🖥 server.js ───── scan AI ──────► WebSocket broadcast │ │ │ GET /api/sms/latest (polling) │ └────────────────────────────────────────► 🖥 Dashboard (halaman ini) ↳ Popup peringatan muncul!
1
Install Node.js & dependensi
Pastikan Node.js versi 18+ sudah terinstall di komputer/server Anda.
node --version # harus v18+ npm install # jalankan di folder server
2
Buat file konfigurasi .env
Salin file .env.example menjadi .env dan sesuaikan nilainya.
cp .env.example .env # Edit .env: PORT=3000 WS_PORT=3001 API_KEY=ganti-dengan-key-rahasia-anda
3
Jalankan server
npm start # Output: # ✅ HTTP : http://localhost:3000 # ✅ WS : ws://localhost:3001
4
Hubungkan Dashboard ke Server
Klik tombol ⚙ API di header dashboard, lalu isi pengaturan berikut:
Mode : WebSocket (Real-time) URL : ws://localhost:3001 API Key : seniorshield-secret-key
Klik 💾 Simpan & Hubungkan → titik hijau akan menyala.
5
Test pertama
Klik tombol 🧪 Uji di connection bar — popup peringatan akan muncul jika pesan berbahaya terdeteksi.
# Atau test via terminal: curl -X POST http://localhost:3000/api/sms/test

📡 Dokumentasi Endpoint API

Semua endpoint menggunakan format JSON. Endpoint yang memerlukan autentikasi harus menyertakan API Key di header.

🔑 Autentikasi:
Sertakan header berikut untuk endpoint yang dilindungi:
Authorization: Bearer YOUR_API_KEY
POST /api/sms/incoming 🔒 Auth
Menerima SMS baru dari Android/forwarder. Server langsung menganalisis dan broadcast ke WebSocket.
curl -X POST http://localhost:3000/api/sms/incoming \ -H "Content-Type: application/json" \ -H "Authorization: Bearer seniorshield-secret-key" \ -d '{ "sender": "+6281234567890", "body": "Verifikasi akun sekarang: http://wa-verify.xyz", "timestamp": "2025-01-01T10:00:00Z" }' # Response: { "success": true, "id": "uuid-xxx", "riskLevel": "high", "score": 75, "status": "HIGH RISK — Kemungkinan Phishing", "keywords": ["verifikasi akun", ".xyz"] }
POST /api/sms/test ✓ Public
Kirim pesan uji tanpa API Key. Berguna untuk testing koneksi dashboard.
curl -X POST http://localhost:3000/api/sms/test # → Memilih pesan uji acak (berbahaya/aman)
GET /api/sms/latest 🔒 Auth
Ambil pesan baru sejak timestamp tertentu. Digunakan dashboard dalam mode HTTP Polling.
GET /api/sms/latest?since=2025-01-01T10:00:00Z&limit=50&risk=high # since : ISO timestamp (default: 1 jam lalu) # limit : maks pesan (default: 50) # risk : filter high | medium | low | all
GET /api/status ✓ Public
Cek status server. Cocok untuk health check dan monitoring uptime.
curl http://localhost:3000/api/status # Response: { "status": "online", "uptime": "1234.5s", "messages": 42, "wsClients": 2 }
GET /api/stats 🔒 Auth
Statistik total deteksi per kategori risiko.
DEL /api/sms/clear 🔒 Auth
Hapus semua log pesan dari memori server. Statistik juga direset.

⚡ WebSocket Protocol

Server push pesan baru ke semua klien yang terhubung secara real-time.

// Hubungkan ke WebSocket: const ws = new WebSocket('ws://localhost:3001?token=YOUR_API_KEY'); ws.onopen = () => console.log('Terhubung!'); ws.onmessage = (event) => { const messages = JSON.parse(event.data); // array messages.forEach(msg => { console.log(msg.sender, msg.riskLevel, msg.score); // → "+6281234567890" "high" 75 }); }; // Format pesan dari server: { "id": "uuid", "sender": "+6281234567890", "body": "isi SMS", "riskLevel": "high", // high | medium | low "score": 75, // 0-100+ "status": "HIGH RISK — ...", "keywords": ["otp", ".xyz"], "timestamp": "ISO8601", "isTest": false }

📱 Integrasi SMS Android

Ada 3 cara mengirim SMS dari Android ke server SeniorShield. Pilih yang paling mudah sesuai kebutuhan.

📲
SMS Forwarder App
Termudah. Install app, tambah rule forwarding. Tidak perlu coding.
⚙
Tasker + AutoHttp
Flexible. Bisa filter berdasarkan nomor, jam, dll.
📦
Android App Custom
Paling powerful. Kontrol penuh dengan Java/Kotlin.
A
Cara A — SMS Forwarder (Paling Mudah)
Install SMS Forwarder dari Google Play Store.
Konfigurasi di SMS Forwarder: ──────────────────────────────────── URL : http://IP-SERVER:3000/api/sms/incoming Method : POST Headers : Authorization: Bearer seniorshield-secret-key Content-Type: application/json Body : { "sender": "{sender}", "body": "{body}", "timestamp": "{timestamp}" }
💡 Ganti IP-SERVER dengan IP komputer Anda di jaringan lokal (cek dengan ipconfig / ifconfig)
B
Cara B — Tasker + AutoHttp Plugin
Install Tasker + plugin AutoHttp dari Play Store.
Tasker Profile: Event → Phone → Received Text Task Actions: 1. AutoHttp URL : http://IP-SERVER:3000/api/sms/incoming Method : POST Header : Authorization: Bearer seniorshield-secret-key Body : {"sender":"%SMSRN","body":"%SMSRB"}

☁ Deploy ke Cloud

Agar server bisa diakses dari mana saja (tidak hanya localhost), deploy ke platform cloud. Berikut tiga pilihan gratis/murah.

🚂
Railway (Recommended — Free tier tersedia)
# 1. Install Railway CLI npm install -g @railway/cli # 2. Login & deploy railway login railway init railway up # 3. Set environment variables di Railway dashboard: # PORT=3000 # WS_PORT=3001 # API_KEY=key-rahasia-anda # 4. Dapatkan URL public: # https://yourapp.railway.app
Setelah deploy Railway, ubah URL di dashboard:
HTTP: https://yourapp.railway.app
WebSocket: wss://yourapp.railway.app
🐳
Docker (Self-hosted VPS)
# Build & run dengan Docker Compose: docker compose up -d # Atau manual: docker build -t seniorshield-api . docker run -d -p 3000:3000 -p 3001:3001 \ -e API_KEY=key-rahasia \ seniorshield-api
⚡
Render.com (Free tier tersedia)
⚠ Penting setelah deploy: WebSocket di cloud biasanya menggunakan wss:// (bukan ws://) dan HTTP menggunakan https://. Sesuaikan URL di pengaturan API dashboard.

🔒 Keamanan API

Panduan mengamankan server SeniorShield agar tidak dapat diakses sembarangan orang.

1
Ganti API Key default
Jangan pernah gunakan key default di production!
# Generate key acak yang kuat: node -e "console.log(require('crypto').randomBytes(32).toString('hex'))" # Contoh output: a3f8e2b1c9d4... # Simpan di .env: API_KEY=a3f8e2b1c9d4...
2
Aktifkan HTTPS / WSS
Untuk production, selalu gunakan HTTPS agar data SMS tidak bocor saat transit. Platform cloud (Railway, Render) otomatis menyediakan HTTPS.
Jika self-host di VPS, gunakan nginx + Let's Encrypt sebagai reverse proxy dengan SSL.
3
Batasi IP (Opsional)
Jika Android dan server di jaringan yang sama, batasi akses endpoint hanya dari IP lokal.
# Di .env tambahkan: ALLOWED_IP=192.168.1.100 # IP Android Anda
4
Jangan commit .env ke Git
🚨 Data SMS bersifat sensitif! Pastikan server hanya bisa diakses oleh perangkat yang berwenang. Jangan expose port 3000/3001 ke internet tanpa proteksi.
